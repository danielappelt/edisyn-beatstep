Oberheim Matrix 1000

Single Patch Editor

This editor will work with the Oberheim Matrix 1000. Though they are very similar, the Matrix 6/6r will probably not work with this patch editor (yet). Someone will have to modify it (note: I'm presently using the patch name to extract the bank number; that won't work on the 6/6r).

Communicating with Edisyn Make sure that Memory Protect is off (see page 36 of the manual).

About the Matrix 1000 Menu

The Matrix 1000 does not store patch names. Normally when it sends Edisyn a patch, the patch name will look something like BNK4: 72, indicating the bank and patch number. Edisyn can instead display the patch name from the Oberheim Matrix 1000 documentation: just select Use Classic Patch Names.

Similarly, if you would like to load a patch from memory, you can look it up by patch name in the Matrix 1000 menu.

Suggestions

  • If you add a lot of random noise (like >50%) to the Matrix 1000, it will often fall silent. This is almost always due to randomization of either the Amplifier section or the Amplifier Envelope (Envelope 2). I suggest that you first edit the mutation parameters to remove these sections as mutation candidates.

Gotchas

  • The original ROM (1.1.1) for the Matrix 1000 can't handle real-time updates very fast: it clogs up quickly. If you have one of these machines, you should turn off Sends Real Time Changes in the MIDI menu. Alternatively you can upgrade your ROM to a better one. There are two options, 1.1.6 and 1.2.0. I suggest 1.2.0, which I think is somewhat better. If you are in Europe, you can buy the ROM (it's quite cheap) through Untergeek:
    • http://www.untergeek.de/howto/oberheim-matrix-1000/oberheim-matrix-1000-firmware-v1-20/

    If you are in the USA, you can buy the ROM through Bob Grieb (the author):

    • http://tauntek.com/contact-information.htm

    Note that even with a new ROM, the Matrix 1000 still will be slightly pokey in keeping up with certain parameters. But it's far better than the original ROM.

  • The patch editor allows you to give patches a name. This is backwards-compatible with the 6/6r. However, when you store a patch in the Matrix 1000, your name is not stored. Instead if you request the patch again, it'll come back with the name BNKb: nn, where b is the bank number, and nn is the two-digit patch number.
